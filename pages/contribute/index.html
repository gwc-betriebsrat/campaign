GWC — Contributing Guide

Everything you need to make changes to the GWC zLife pages.

The GWC pages you see on zLife are not managed through zLife itself. Behind the scenes, they are HTML files stored in a GitHub repository. zLife is simply displaying those files inside a frame — like a window into the repo.

GitHub is a platform for storing and collaborating on files, similar to a shared drive, but with a full history of every change ever made and the ability to instantly undo anything.

The repository is at: github.com/gwc-betriebsrat/campaign

To make a change to what people see on zLife, you edit the relevant file in that repository directly on the main branch and save it (this is called a "commit"). No deployment step, no approval process — the zLife pages update automatically within about 60 seconds.

The rest of this guide explains how to get access and how to make those edits.

1. The repository

All files that make up the zLife pages live here:

https://github.com/gwc-betriebsrat/campaign

When you save a change (called a "commit"), the zLife pages update automatically within about 60 seconds.

2. Getting access

  1. Create a free account at github.com if you don't have one
  2. Send your GitHub username to the team admin — they'll add you as a collaborator
  3. Once added, you can edit files directly in the browser

3. Project structure

The repo looks like this:

gwc/
├── shared/
│   └── _shared.css          ← shared styles (colours, fonts, layouts)
└── pages/
    ├── who-we-are/          ← candidate list, story, regions, ERGs
    ├── mission/             ← mission statement + 3 principles (EN + DE)
    ├── commitment/          ← 4 commitments + 10 beliefs
    ├── values/              ← 6 value cards
    ├── join/                ← volunteer + feedback forms
    ├── accomplishments/     ← track record (work in progress)
    └── contribute/          ← this page

Each page is a single index.html file inside its folder.

4. Making an edit

  1. Go to the repo and navigate to the file you want to change (e.g. pages/commitment/index.html)
  2. Click the pencil icon ✏️ in the top-right of the file view
  3. Make your change
  4. Scroll to the bottom — write a short description of what you changed
  5. Click Commit changes

To check it's live: wait ~60 seconds, then hard-refresh the page (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac).

5. HTML basics

You only need to know a few patterns:

Paragraph

<p>Your text here.</p>

Heading

<h2>Section title</h2>
<h3>Sub-heading</h3>

Bold / red text

<strong>bold</strong>
<strong style="color:var(--red);">red and bold</strong>

Link

<a href="https://example.com" target="_top">link text</a>

Always use target="_top" on links so they open in the full browser tab, not inside the iframe.

List

<ul>
  <li>First item</li>
  <li>Second item</li>
</ul>

Rule of thumb: only change text between tags. Don't rename, delete, or reorder the tags themselves.

6. Page-by-page guides

Each page folder has a README.md file with specific instructions for that page:

  • who-we-are — candidates, story text, regions, ERGs
  • mission — mission statement, principles (EN + DE)
  • commitment — commitments, beliefs, contact email
  • values — value cards
  • join — form dropdowns, checkboxes
  • accomplishments — timeline milestones

7. Updating the candidate list

The candidate list is in pages/who-we-are/index.html and appears twice — once in the English section and once in the German section. Always update both.

The source of truth is archive/Candidates Details - v3.csv. The rules:

  • Show on page: OK to share their name = YES
  • Clickable card with photo: Zlife published = YES and Link to introduction is filled
  • Photo URL: right-click the photo on the zLife article → Inspect → copy the src from the <img> tag

Card without article (initials, not clickable)

<div class="team-card">
  <div class="team-avatar">AB</div>
  <div class="team-name">Alex Bauer</div>
  <div class="team-role">Senior Engineer</div>
</div>

Card with article (photo, clickable)

<a class="team-card" href="ARTICLE_URL" target="_top">
  <div class="team-avatar"><img src="PHOTO_URL" alt="Alex Bauer" /></div>
  <div class="team-name">Alex Bauer</div>
  <div class="team-role">Senior Engineer</div>
  <div class="team-link-hint">Get to know me ↗</div>
</a>

8. Reverting a change

  1. Go to the file in the repo
  2. Click History (top-right of the file view)
  3. Find the last good version and click on it
  4. Click Browse files → navigate to the file → click ✏️ → copy the old content back in → commit

Nothing is ever permanently deleted — every version is stored.
